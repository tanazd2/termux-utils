## 1. How to Contribute

There are lots of ways you can contribute:

- 🐛 Fix bugs  
- ✨ Add new features  
- 📄 Improve documentation  
- 🧪 Add tests  
- 💬 Suggest ideas or improvements

Even small contributions are welcome!

---

## 2. Getting Started

1. **Fork** this repository  
2. Clone it locally:
   ```bash
   git clone https://github.com/yourusername/termux-utils.git
   ```
3. Create a branch for your work:
   ```bash
   git checkout -b feature/your-feature
   ```

---

## 3. Development Workflow

### Branch Naming
Use clear, descriptive branch names. For example:

```
feature/add-logging
fix/typo-in-readme
docs/update-contributing
```

### Keep your fork up-to-date
```bash
git fetch upstream
git merge upstream/main
```

---

## 4. Reporting Issues

When you open an issue, include:

- A descriptive title  
- Steps to reproduce the problem  
- Expected vs. actual behavior

This makes it easier for maintainers to help you.

---

## 5. Pull Request Guidelines

When submitting a PR:

1. Open a Pull Request from your fork → this repository  
2. Provide a clear title and description explaining **why** the change matters  
3. Reference any related issues:  
   ```
   Closes #123
   ```
4. Keep your changes logically grouped

---

## 6. Code Style & Standards

To make reviews easier and consistent:

- Follow the existing code style (indentation, naming, formatting)  
- Write clear, readable code  
- Keep functions focused and small where possible

---

## 7. Tests

If applicable:

- Add tests for new features or bug fixes  
- Make sure existing tests pass

---

## 8. Communication & Etiquette

- Be respectful and patient  
- Respond politely to feedback  
- Ask questions if you’re unsure — the community is here to help

We strive for a welcoming and positive environment for everyone.
